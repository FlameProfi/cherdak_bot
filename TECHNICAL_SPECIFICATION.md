# Техническое задание: Система лояльности Telegram-бот для Fusion POS

## 1. Общие сведения
Данный проект представляет собой Telegram-бота, интегрированного с облачной POS-системой Fusion POS. Бот предназначен для автоматизации системы лояльности гостей кальянной, предоставляя им удобный доступ к своей виртуальной карте, текущему статусу и накоплениям.

## 2. Логика уровней лояльности
Скидка рассчитывается на основе общей суммы покупок клиента (накопительный итог):

| Уровень | Сумма покупок (от) | Скидка |
| :--- | :--- | :--- |
| **Новичок** | 0 руб. | 5% |
| **Ценитель** | 15 000 руб. | 10% |
| **Мастер** | 40 000 руб. | 15% |
| **Легенда** | 80 000 руб. | 20% |

Максимально возможная скидка в системе: 20%.

## 3. Сценарии взаимодействия

### 3.1. Взаимодействие для гостя
1. **Первый вход и верификация:**
   - Гость запускает бота и нажимает кнопку «Поделиться контактом».
   - Бот получает номер телефона и проверяет его наличие в базе Fusion POS через API.
   - Если клиент найден — подтягиваются его данные (ID, история покупок).
   - Если клиент не найден — создается новый клиент в базе Fusion POS.
2. **Главный экран «Моя карта»:**
   - Отображает: Имя гостя, Текущий статус (название уровня), Текущий % скидки.
   - Прогресс: «Сколько осталось потратить до следующего уровня».
   - Идентификация: QR-код (содержит ID клиента или номер телефона) и цифровой код.
3. **Дополнительные функции:**
   - Кнопка «Меню» (ссылка на электронное меню).
   - Кнопка «Забронировать стол» (ссылка или переход в диалог).
   - Кнопка «Оставить отзыв».

### 3.2. Взаимодействие для персонала (кальянщика)
- Кальянщик работает исключительно в терминале Fusion POS.
- При расчете стола кальянщик сканирует QR-код из телефона гостя или вводит номер телефона в поле «Клиент».
- Текущая скидка применяется автоматически средствами Fusion POS.
- После закрытия чека Fusion POS отправляет Webhook боту.

### 3.3. Взаимодействие для владельца
- Доступ к админ-меню по Telegram ID.
- **Рассылки:** Возможность отправки сообщений определенным группам клиентов (например, только «Новичкам») или всем пользователям.
- **Статистика:** Просмотр количества зарегистрированных пользователей и их распределение по уровням.

## 4. Технические требования
- **Язык реализации:** Node.js.
- **База данных:** SQLite (хранение связки Telegram ID <-> Fusion ID).
- **Интеграция:** Fusion POS API v4 (REST).
- **Уведомления:** Использование Webhooks от Fusion POS для мгновенных Push-уведомлений о покупках.
